1.1. Problem Statement¶

At Kitty of Angels, a 501(c)(3) non-profit devoted to supporting homeless cats and kittens in Los Angeles, understanding our foster system's intricacies is crucial to our mission. My primary objective in analyzing the 'intakes' and 'outcomes' datasets is to determine how factors such as a cat or kitten's sex and breed correlate with its length of stay in our foster system. By delving deep into these variables, I aim to uncover patterns or insights that could help optimize our operations, ensuring that every feline under our care finds a loving home in the shortest possible time.

# Reading the data from the html file.
with open('Foster_Outcomes_Shelterluv.html', 'r', encoding='utf-8') as f:
    html_content = f.read()

# Parsing the html content.
soup = BeautifulSoup(html_content, 'html.parser')

# Extracting the second table, which contains the appropriate data.
tables = soup.find_all('table')
data_table = tables[1]

# Extracting the headers and rows from the second table.
header = [th.get_text().strip() for th in data_table.find('tr').find_all('th')]
rows = data_table.find_all('tr')[1:]  # Exclude header row

# Converting the extracted data into a DataFrame using the headers and rows.
df_outcomes_raw = pd.DataFrame([[td.get_text().strip() for td in row.find_all('td')] for row in rows], columns=header)

# Saving the raw DataFrame to 'outcomes_raw.csv'.
df_outcomes_raw.to_csv('outcomes_raw.csv', index=False)

2. Assess data¶

Assessing Quality Issues in the 'intakes'¶

Quality Issue 1: Validity¶

Issue:

  • Some columns in the dataset contain sensitive or private information that could compromise the privacy of individuals or entities. These columns are 'Intake By', 'Intake From (ID)', and 'Intake From Name'. Furthermore, given the number of unique values in 'Intake From Name', there are many individuals whose privacy should be safeguarded. If the number were smaller, it might be viable to contact them for persmission. Another issue concerning validity is the data type of the 'Intake Date' column, which is set as 'object' and will be changed to 'datetime64[ns]'.

Justification:

  • Solving this issue ensures adherance to ethical data handling practices and also ensures the dataset's validity for public use. Furthermore, changing the data type of the 'Intake Date' will allow for deeper analysis when the 'intakes' and 'outcomes' dataframes are merged.
